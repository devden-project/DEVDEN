# 요구사항 분석 문서
**프로젝트 주제** : 블로그와 커뮤니티 결합 사이트 개발  
**프로젝트 관리자** : 고진혁  
**작성일** : 2024년 6월 23일  
**버전** : 1.0.0
## 문서 개요
### 목적
이 문서는 사이트 개발을 위한 요구사항을 명확히 정의하고, 프로젝트의 성공적인 수행을 위해 필요한 정보 제공
### 배경
* IT 정보와 개인 개발 이야기, 학습 정보를 공유하기 위한 플랫폼
* 사용자들이 자유롭게 소통하고 정보를 교환할 수 있는 공간을 제공하는 것을 목표로 함
## 시스템 개요
### 시스템 설명
* 사용자들이 게시글을 작성하고 댓글을 통해 소통할 수 있는 웹 플랫폼
* 자신의 블로그에 학습내용이나 IT 트랜드 등 다양한 정보를 게시 가능하며, 커뮤니티를 통한 QNA, 기타 이야기 등을 게시 및 검색 가능
* 사용자는 자신의 프로필을 관리하고, 다른 사용자를 팔로우할 수 있으며, 실시간 채팅 및 쪽지 사용 가능
* 다양한 카테고리별 정보를 검색하고 열람 가능하며 북마크를 통한 열람글 관리 가능
### 주요 기능
* 사용자 관리
  * 회원가입 및 로그인
  * 비밀번호 재설정
  * 프로필 관리
  * 회원정보 수정
  * 회원 탈퇴
* 게시글 관리
  * CRUD(게시글, 댓글)
  * 좋아요와 팔로우 및 북마크 가능
* 카테고리 관리
  * IT 정보, 개인 개발 이야기, 학습 정보, QNA 등
  * 팔로우한 사용자의 게시글 목록 보기
  * 북마크한 게시글의 목록 보기
  * 카테고리별 게시글 목록 보기
  * 블로그 게시글의 주제 설정
* 검색
  * 키워드로 게시글 검색
  * 사용자 검색
* 일대일 통신
  * 실시간 채팅
  * 쪽지
## 기능 요구사항
### 사용자 관리
#### [FR-001] 회원가입 기능
* **설명** : 사용자가 아이디, 이메일, 비밀번호를 입력하여 회원가입을 할 수 있어야 함
* **우선순위** : P0
* **요구사항 상세** :
  * 이메일 중복 체크
  * 비밀번호 최소 8자 이상, 특수문자 포함
  * 회원가입 완료 후 인증 이메일 발송
#### [FR-002] 로그인 기능
* **설명** : 사용자가 아이디와 비밀번호를 입력하여 로그인 할 수 있어야 함
* **우선순위** : P0
* **요구사항 명세** :
  * 아이디와 비밀번호 검증
  * 로그인 실패 시 에러 메시지 표시
  * 로그인 성공 시 사용자 세션 생성
#### [FR-003] 비밀번호 재설정
* **설명** : 사용자가 필요한 경우 이메일 인증을 통한 비밀번호 재설정 할 수 있어야 함
* **우선순위** : P0
* **요구사항 명세** :
  * 개인정보 수정 페이지에서 변경
  * 비밀번호 찾기 페이지에서 이메일 인증을 통한 변경
#### [FR-004] 프로필 관리
* **설명** : 사용자가 자신을 소개할 수 있는 프로필을 만들 수 있어야 함
* **우선순위** : P1
* **요구사항 명세** : 
  * 프로필 사진 등록
  * 자신의 깃허브나 블로그 주소 표시
  * 커리어(경력 및 수상내역 등) 작성
  * 자신의 기술 스택 작성
  * 소개글 작성
  * 회원정보에 등록한 전화번호나 이메일 등의 개인 연락 수단 등록
#### [FR-005] 회원정보 수정
* **설명** : 사용자가 자신의 프로필 혹은 개인 정보를 수정할 수 있어야 함
* **우선순위** : P1
* **요구사항 명세** :
  * 프로필 및 블로그 공개 여부
  * 이메일, 전화번호 등 수정 및 추가 여부
  * 비밀번호 변경 여부
#### [FR-015] 회원 탈퇴
* **설명** : 사용자가 자신의 회원상태를 탈퇴할 수 있어야 함
* **우선순위** : P1
* **요구사항 명세** :
  * 회원탈퇴 시 ROLE을 회원탈퇴로 표시
  * 회원정보 중 민감정보를 제외한 최소한의 정보만 남김
  * 해당 회원이 남긴 포스트 혹은 댓글들의 유지여부 확인
  * 회원탈퇴 철회 혹은 재가입 시 해당 사용자가 계정을 복구할 수 있도록 함
### 게시글 관리
#### [FR-006] CRUD(게시글, 댓글)
* **설명** : 사용자가 자신의 게시글 및 댓글을 관리할 수 있어야 함
* **우선순위** : P0
* **요구사항 명세** :
  * 게시글의 경우 markdown 기반의 문서로 작성 가능
  * 게시글 작성 시 프리뷰를 볼 수 있는 기능 제공
  * 자신의 게시글 및 댓글에 대한 CRUD 가능
  * 타인의 게시글 및 댓글에 대한 업데이트 및 삭제 통제
  * 게시글의 경우 공개 및 비공개 설정
  * 댓글의 경우 공개 및 해당 게시글 작성자에게만 공개 설정
#### [FR-007] 좋아요와 팔로우 및 북마크
* **설명** : 사용자가 해당 게시글에 대한 좋아요, 팔로우, 북마크를 관리할 수 있어야 함
* **우선순위** : P2
* **요구사항 명세** :
  * 좋아요는 사용자당 1번만 가능
  * 게시글 작성자를 팔로우 시 팔로우해둔 사용자 리스트를 제공
  * 게시글을 북마크 시 북마크해둔 게시글 리스트를 제공
  * 좋아요, 팔로우, 북마크는 사용자가 추가 및 삭제가 가능
### 카테고리 관리
#### [FR-008] IT 정보, 개인 개발 이야기, 학습 정보, QNA 등
* **설명** : 다른 설정 필요없이 사용자가 볼 수 있는 카테고리
* **우선순위** : P1
* **요구사항 명세** :
  * 글 작성 시 해당 카테고리 지정 가능
  * 블로그에서 글 작성 시 해당 카테고리를 지정해두면 커뮤니티의 글 목록에서 조회 가능
  * 블로그에서 글 작성 시 해당 카테고리를 지정하지 않으면 커뮤니티의 글 목록에서 조회 불가
#### [FR-008] 팔로우, 북마크 게시글 목록 보기
* **설명** : 사용자가 지정한 특정 게시글 및 사용자의 글만 모아볼 수 있는 카테고리
* **우선순위** : P2
* **요구사항 명세** :
  * 특정 글을 북마크할 경우 북마크된 글만 모아볼 수 있는 카테고리 제공
  * 특정 사용자를 팔로우할 경우 팔로우한 사용자의 글만 모아볼 수 있는 카테고리 제공
#### [FR-009] 카테고리별 게시글 목록 보기
* **설명** : 사용자가 특정 카테고리의 글들만 모아서 볼 수 있도록 함
* **우선순위** : P1
* **요구사항 명세** :
  * 전체 게시글을 볼 수 있는 기능과 특정 카테고리의 글들만 볼 수 있는 기능을 제공
#### [FR-010] 블로그 게시글의 주제 설정
* **설명** : 커뮤니티의 카테고리 외의 자신의 블로그에서 사용할 개인 카테고리 설정
* **우선순위** : P0
* **요구사항 명세** :
  * 커뮤니티의 지정된 카테고리 외의 자신의 블로그에서만 사용할 개인 카테고리 제공
  * 개인 카테고리는 사용자가 임의로 설정 가능
  * 사용자가 지정한 블로그 카테고리별 글 목록 조회 및 관리 가능
### 검색
#### [FR-011] 키워드로 게시글 검색
* **설명** : 카테고리의 목록 혹은 전체글의 목록을 조회한 경우, 키워드를 통한 게시글 검색 가능
* **우선순위** : P2
* **요구사항 명세** :
  * 게시글의 제목 혹은 내용에 포함된 내용을 키워드로하여 조회가 가능
  * 게시글의 제목 혹은 내용에서 글의 도입부, 중간, 맺음부 등의 위치에 상관없이 조회 가능
#### [FR-012] 사용자 검색
* **설명** : 특정 사용자를 조회하기 위해 사용자의 아이디를 통한 검색 가능
* **우선순위** : P2
* **요구사항 명세** :
  * 특정 사용자를 검색하여 블로그나 프로필 등을 조회 가능
### 일대일 통신
#### [FR-013] 실시간 채팅
* **설명** : 특정 사용자와 실시간 채팅 기능 제공
* **우선순위** : P2
* **요구사항 명세** :
  * 특정 사용자에게 실시간 채팅 요청을 보내고 수락 시 진행
  * 대화 내용은 db에 최대 1000건까지 저장되며, 오래된 대화부터 삭제
  * 대화가 이루어지는 채팅방이 1달 이상 미사용인 경우 db의 대화내용 삭제
  * 대화내용을 로컬 스토리지에 저장되게 하여 db이외에 대화내역 보관
#### [FR-014] 쪽지
* **설명** : 특정 사용자와 쪽지 기능 제공
* **우선순위** : P2
* **요구사항 명세** :
  * 특정 사용자에게 쪽지를 보내고 해당 쪽지에 대한 답장을 받을 수 있도록 함
  * 수신자 별 쪽지를 모아볼 수 있도록 함
  * 최대 수신자 별 1000건까지 저장되며, 오래된 쪽지부터 삭제
  * 쪽지가 전송된 후 3개월이 지난 경우 db의 쪽지내용 삭제
  * 쪽지내용을 로컬 스토리지에 저장되게 하여 db이외에 쪽지내역 보관
## 비기능 요구사항
### 성능 요구사항
#### [NFR-001] 모든 페이지 로딩 시간
* **설명** : 모든 페이지는 3초 이내에 로드
* **우선순위** : P0
### 보안 요구사항
#### [NFR-002] 인증 및 권한 부여
* **설명** : OAuth2.0 또는 JWT를 사용하여 안전한 인증 및 권한 부여 구현
* **우선순위** : P0
#### [NFR-003] 악의적 요청 방지
* **설명** : XSRF-TOKEN을 이용하여 사용자가 원치 않는 악의적 요청 방지
* **우선순위** : P0
#### [NFR-004] 저장 데이터 암호화
* **설명** : db에 저장되는 모든 민감 데이터 암호화
* **우선순위** : P0
#### [NFR-005] 전송 중 데이터 암호화
* **설명** : HTTPS사용으로 전송 데이터를 암호화 하고 TLS를 통한 데이터 전송 보호
* **우선순위** : P1
